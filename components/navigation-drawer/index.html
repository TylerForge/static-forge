Navigation drawer

Navigation drawers provide access to destinations in your app.

Interactive demo

Client-Facing Terminology

Do not use the term drawer in client-facing documentation. Refer to this as a menu when it is temporary and fully collapsible. Refer to it as a sidebar when it persists on the page.

Examples:

  • Click Drafts in the menu.
  • Select an option in the sidebar.

Guidance

The navigation drawer slides in from the left and contains the navigation destinations for your app. The drawer spans the height of the screen, with everything behind it visible but darkened by a scrim.

The navigation drawer follows the specs and metrics for lists.

Tyler uses 4 drawer types:

  • Permanent
  • Dismissible
  • Mini variant
  • Modal

Drawer types

Permanent

Recommended default for desktop; not suitable for mobile due to limited screen size.

A permanent navigation drawer allows for interaction with both screen content and the drawer at the same time. 

When users need to switch destinations frequently (and screen size allows), a permanently visible drawer can be used.

The permanent navigation drawer is always visible and pinned to the left edge, at the same elevation as the content or background. It cannot be closed.

Dismissible

Recommended to be used in desktop; not suitable for mobile due to limited screen size.

A dismissible drawer can be used when users are likely to focus on screen content and require less frequent access to its navigation destinations.

Modal

In a responsive layout grid, at a defined minimum breakpoint of at least 600dp width, a standard drawer should be replaced with a modal drawer.

A modal navigation drawer blocks interaction with the rest of an app’s content with a scrim. It is elevated above most of the app’s UI and doesn’t affect the screen’s layout grid.

Modal navigation drawers are primarily for use on mobile where screen space is limited, and can be replaced by standard drawers on tablet and desktop.

Anatomy

Navigation drawers contain a list embedded within a sheet. They can be enhanced with headers and dividers to organize longer lists.

1.Container

2. Header (Required for public facing apps)

The header contains an avatar, a citizen’s name, and their primary identifier (may be a username, email address, address, or account number). Header information should be limited to the user’s name and up to two lines of supporting text. 

3. Divider (optional)

4. Active text overlay

5. Active text

6. Inactive text

7. Subtitle

Subtitles use –body2 (text at 13px) and –mdc-theme-text-secondary-on-background (black at 54%).

8. Scrim (modal only)

A scrim should not be used for permanent drawers.

Best Practices

Destination labels

Do.

Navigation drawers can use text labels without icons.

Do.

Keep text labels concise, but truncate them if they extend beyond the container width.

Don’t.

Don’t wrap label text.

Don’t.

Don’t shrink text size in order to fit a text label on a single line.

Destination iconography

Don’t.

Don’t use the same icon to represent different primary destinations.

Don’t.

Don’t apply icons to some destinations and not others. Icons should be used for all destinations, or none.

Development

Why isn't my TCW component rendering?

Each custom element HTML tag must be registered with the browser so that it knows how to instantiate the component when it parses the DOM.

Did you remember to define the component(s) when your application is loaded?

There are convenience functions that are exported from the TCW library that will ensure that the elements are properly defined using the customElements API.

See the getting started guide for usage information: Getting Started - Angular

What's the current version of TCW?
The current TCW version is always displayed on the Forge homepage
If I consume the TCW components, what happens when there are upgrades?

The TCW library is available via NPM, and strictly follows semantic versioning. Breaking changes will be limited to major versions. There is a CHANGELOG posted with each release that defines the fixes, features, and breaking changes that pertain to each version.

When installing a new version, if there are no breaking changes, the upgrade path should be seamless.

What frameworks can use TCW?

TCW Web Components are built for the browser platform which means they are framework-agnostic. They integerate seamlessly with all popular JavaScript frameworks including Angular, React, Vue... etc.

For instance, in Angular you can bind to the element properties just as you would any other built-in HTML element.

Who do we reach out to for creating new components?

Please use the Forge feedback & support page for all requests, questions, and general feedback.

Why isn't my binding working in Angular on a TCW component?

This is typically due to binding to the attribute instead of the JavaScript property. For example to set the title text of the omnibar component, you should use this format [titleText]="title" instead of [title-text]="title". Angular uses the JavaScript API of all HTML elements to bind values.

When I click the select I can't see the dropdown?

There could be a few different reasons why the dropdown doesn't show:

  • If you're using template-based tcw-option elements, be sure they are rendered on the page at runtime by inspecting the DOM with your dev console. If there are no options, the dropdown won't display. The same thing goes for if you're using the options property, ensure that it has items in it.
  • When clicking on the select, check your dev console to ensure that the tcw-popup element is being appended to the bottom of the body, as well as ensure that the popup has your expected options within the list. If this is not there, then you likely have a problem with the first issue above, or your select component was defined with the browser.
  • The only other problem here could be a z-index issue. If the tcw-popup element is being placed correctly in the DOM, then try setting the --tyl-z-index-popup CSS custom property to a higher value. This can be applied globally to your application. The z-index ranges are exposed to offer the ability to adjust the floating elements to play nice with other CSS libraries, such as Angular Material.

Questions

What is Forge, exactly?

Tyler Forge is a process, not a project. Contributions to our design system will be welcomed from all product teams and will be curated by the Forge Advisory Board. Our design system captures what we have accomplished, and relies on the product teams for future growth – through their ideation, exploration and contribution.

Okay… sure…but, what is Forge, exactly?

The website you are on is one piece of our overall design system, Forge. This site is intended to be a living, breathing representation of design documentation for our layouts, patterns, and components (components together = a pattern, patterns and/or components together are part of an overall page or app layout).

 

Forge, however, includes this site as well as our overall process on the UX team–our alignment on how our products support an overall vision and brand for Tyler, our process for working with teams through end-to-end design and research for products, and all the steps in between. Forge, a design system, is a complete set of design standards, documentation, and principles along with the toolkit (UI patterns and code components) to achieve those standards.

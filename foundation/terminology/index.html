Terminology

Design System – the single source of truth which groups all the elements that will allow the teams to design, realize and develop a product.

Layout – the way in which the parts of a page or screen are arranged or laid out. Examples: a 3-column layout versus left navigation with a main content window.

Responsive Design – an approach to web page creation that makes use of flexible layouts, flexible images and cascading style sheet media queries. The goal of responsive design is to build web pages that detect the user’s screen size and orientation and change the layout accordingly.

User Testing – a technique used in user-centered design to evaluate a product or feature by testing it on users. This can be seen as an irreplaceable usability practice, since it gives direct input on how real users use the system.

Mockups – a visual way of representing a product or screen design. Like a wireframe, it is not clickable.  As opposed to a wireframe, a mockup is either mid- or high-fidelity.

Persona – a representation of a type of customer or user. Personas answer the question, “who are we designing for?” and they help to align strategy and goals to specific user groups.

Pattern – can be comprised of 1 or more components, patterns help designers devise workable solutions to common interface problems. Examples: filter and search.

Component – a piece of the user interface with which the user can interact. Examples: a button, a card, a data table.

User Research – focuses on understanding the user, their behaviors, needs, and motivations through observation techniques, task analysis, and other feedback methodologies.

Mockups – a visual way of representing a product or screen design. Like a wireframe, it is not clickable.  As opposed to a wireframe, a mockup is either mid- or high-fidelity.

Prototype – Used to understand and see how a product or application works, what it does and how a user or persona should interact with it. They’re simulations of how a finished product will work and utilized to conduct user testing.

Wireframes – an image or set of images which displays the functional elements of a website or page, typically used for planning a site’s structure and functionality. A wireframe is a low-fidelity presentment of the user interface.

Outcome – the goals of the user, based on their persona and scenario.

Iconography – a collection of illustrations or symbology which are used to represent an interface action, a record status or communicate something visually to the user.

Data Visualization – the representation of information in the form of a chart, diagram, picture, etc.

Accessibility – refers to the practice of building digital content and applications that can be used by a wide range of people, including individuals who have visual, motor, auditory, speech, or cognitive disabilities.

Palette – the range of colors used by a particular product or design.

Menu – a component consisting of navigation options, especially one displayed on screen.

Toolbar – a component consisting of a strip of icons used to perform certain functions within the user interface.

List – a component consisting of a list of items, that do not have named rows or columns.

Scenario – based on a persona or persona group, the fictitious story of a user accomplishing an action or goal via a product. It focuses on a user’s motivations and documents the process by which the user might use the product flow and design.

Accessibility – refers to the practice of building digital content and applications that can be used by a wide range of people, including individuals who have visual, motor, auditory, speech, or cognitive disabilities.

Browser Support – refers to the manner in which a web page looks in different web browsers. The major browsers to ensure your site is compatible include IE, Chrome, Firefox, Opera, Safari on mobile devices.

APIs – An application program interface (API) is code that allows two software programs to communicate with each other. The API defines the correct way for a developer to write a program that requests services from an operating system (OS) or other application.

Typography – the style, arrangement, and appearance of the letters, numbers, and symbols within an interface design.

Card – a piece of material or “card” which typically contain a header and label/data pairs. In some instances the card may be clicked to be expanded to display additional information.

Forms –a web page that allows a user to enter data that is sent to a server for processing.

Data Table – a component consisting of information displayed in tabular form, with named rows and/or columns.

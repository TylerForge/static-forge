Theme

This page documents all theming utilities such as CSS custom properties, classes, and styles. This is all backed by a larger set of Sass mixins, variables, and functions that can be used by applications directly to achieve the desired theme.

A default light theme is provided by default through the TCW stylesheet, but creating a specialized stylesheet is possible through the use of the theming utilities, as well as dark theme support (coming soon!).

Guidance

In the following sections you will find specific implementation details for theming, but if you’d like to learn more about the topics discussed in this page first, please visit the following links:

  • CSS custom properties
  • Sass mixins & variables

The TCW library derives a lot of its theming from Google’s material-components-web (MDC) project under the hood. They provide a good overview of their theme information on their GitHub: https://github.com/material-components/material-components-web/blob/master/docs/theming.md

CSS custom properties

TCW makes heavy use of CSS custom properties. This browser feature allows us to define our own custom CSS style names, and use/apply them across the library and page where necessary. The great thing about CSS custom properties is that we can define as many as we’d like under the :root selector, and have those cascade down through the whole page.

Another important feature of CSS custom properties is that they only pertain to the element they are applied to, and all of the child nodes of that element that reference the property. This allows us to override certain styles on a per-element basis.

For components that use Shadow DOM, CSS custom properties become extremely useful for allowing certain styles to be changed from outside of the shadow root. Styles in the Shadow DOM are encapsulated, and not controllable by the containing parent context. CSS custom properties pierce the Shadow DOM, and allow any style to be adjusted if necessary.

IE 11 does not support CSS custom properties natively, but there is a polyfill available. Alternatively, use standard CSS overrides if needed since it does not natively support Shadow DOM.

The CSS custom properties, for all TCW components that make use of them, are documented on each individual component page.

Below are a few of the most important CSS custom properties that are set on the page when importing the TCW theming styles. Inspect the page of this site to see everything that is available!

Name

Description

--mdc-theme-primary

This contains the primary theme color.

--mdc-theme-secondary

This contains the secondary/accent theme color.

--mdc-theme-tertiary

This contains the tertiary theme color.

--mdc-theme-background

This contains the application background color.

--mdc-theme-surface

This contains the surface color to be used on elements such as cards, dialogs, drawers... etc.

--mdc-theme-surface

This contains the surface color to be used on elements such as cards, dialogs, drawers... etc.

--mdc-theme-error

The color to use for displaying errors.

--mdc-theme-text-primary-on-background

The text color to be used on the background theme color.

To use any CSS custom property as a value for a style, you need to use the var([name]) syntax.

For example:

color: var(--mdc-theme-primary);

Sass variables

Similar to CSS custom properties, there are Sass variables that are available to be imported and used within your project for access to theming (and many other individual style values) at build time.

Below is a brief list of some of the important variables, and where to find them:

Name

Description

$tyl-theme-primary

This contains the primary theme color.

$tyl-theme-secondary

This contains the secondary/accent theme color.

$tyl-theme-tertiary

This contains the tertiary theme color.

$tyl-theme-background

This contains the application background color.

$tyl-theme-success

The success color.

$tyl-theme-warning

The warning color.

$tyl-theme-danger

The error/danger color.

$tyl-theme-info

The info color.

$tyl-omnibar-color

The background color of the omnibar.

All of the above variables (and more) are defined in the TCW theme variables sass file:

https://bitbucket.tylertech.com/projects/TWC/repos/tyler-components-web/browse/src/lib/theme/_variables.scss

Dark theme

The TCW library also exports a dark theme stylesheet. This stylesheet simply overrides the existing CSS custom properties in the default stylesheet, and adjusts their values to match the dark design.

To load the dark stylesheet, you must load it after the default stylesheet. If your application users any static color values in your CSS, you should update the values to use the correct CSS custom properties that are discussed above.

Autocomplete

The autocomplete component will provide the user with a dropdown containing suggested options as they type.

Installation

This component can be installed as an individual npm package via @tyler-components-web/autocomplete

To install the full TCW component library package, see the getting started guide.

Guidance

This is a generic component that is intended to be used around any content that contains an input element. The input element will be detected and attached-to automatically to enable autocomplete functionality.

Usage

This component requires the use of JavaScript to handle the filtering of the data. Whenever the user clicks on the field, or starts typing while it has focus, the filter callback that is configured will be called with the current value that the user has typed. It will also provide the currently selected value if it has one.

Example
arrow_drop_down 
<tcw-autocomplete>
  <tcw-text-field>
    <input type="text" id="input-01" />
    <label for="input-01">Choose statelabel>
    <i slot="trailing" class="material-icons tyl-dropdown-icon" aria-hidden="true">arrow_drop_downi>
  tcw-text-field>
tcw-autocomplete>

Set the filter callback to be used so that you can provide data back to the component to display whenever a filter action is executed by the component.

const autocomplete = document.querySelector('tcw-autocomplete');
autocomplete.filter = (filter, value) => {
  // Return the data as an array of IOption types
  //    - Could call a service here and return a promise to the autocomplete
  //    - Could filter the data and return it directly here.
  //    - If no filter value is provided, return a sub-set of pre-filtered data.
};

The tyl-dropdown-icon class being applied to the trailing icon is what controls the rotation of the icon when the input is focused. This class is available through the TCW theme CSS.

Properties/Attributes

Property

Attributes

Description

multiple

multiple

Controls whether multiple selections are allowed. Default is false.

value

value

Gets/sets the value of the autocomplete.

debounce

debounce

Controls the keypress debounce delay threshold when calling the provided filter function.

highlightFirst

highlight-first

Controls whether the first option in the dropdown is highlighted when visible.

filterOnFocus

filter-on-focus

Controls whether the filter callback and dropdown will be shown when the component receives focus.

allowUnmatched

allow-unmatched

Gets/Sets whether the text entered within the input will be unchanged when a matching option is not found.

popupTarget

popup-target

Gets/sets the selector that will be used to find an element to attach the popup to. Defaults to the input element.

popupClasses

popup-classes

Gets/sets the list of classes to apply to the popup element. Can be a string of space-separated classes.

filter

Sets the filter function. This is required.

optionBuilder

Sets the option template builder callback.

observeScroll

observe-scroll

Toggles whether scroll events are emitted from the dropdown.

observeScrollThreshold

observe-scroll-threshold

A pixel amount that controls when scroll events are emitted when scrolling within the popup.

Events

Name

Values

tcw-autocomplete-change

Emits when the user selects an option from the suggestions.

tcw-autocomplete-scrolled-bottom

Emits when the user scrolls to the bottom of the dropdown. Only if observeScroll is true.

  builder?: AutocompleteOptionGroupBuilder;
  options: IOption[];

type AutocompleteFilterCallback = (filterText: string, value: string | string[] | null) => IOption[] | IOptionGroup[] | Promise;

type AutocompleteOptionBuilder = (option: IOption, filterText: string) => HTMLElement;

App launcher

The app launcher is a convenient navigation component that contains a personal and context-specific map to other parts of the Tyler ecosystem. It is intended to be located and accessed from within the omnibar actions for consistency and familiarity across all applications.

Installation

This component can be installed via the omnibar npm package: @tyler-components-web/omni

To install the full TCW component library package, see the getting started guide.

Guidance

Although the app launcher is available as a standalone component, you will almost always use the convenient tcw-omnibar-app-launcher-button component. This will provide a common mechanism for accessing the app launcher within a popup, along with a common icon to use within the omnibar. More on this component below.

It’s worth noting that the full app-launcher component API is available through the wrapper button component.

Usage

To get started implementing this component into the omnibar, you will want to use the tcw-omnibar-app-launcher-button component that will ensure you are using the correct icon, as well as handle all of the interaction for you. The only requirement on your part is to provide the component an optionsCallback that can be invoked when the app-launcher needs to fetch options to display to the user. This function can simply a be callback function reference that return a promise containing the array of IAppLauncherOption objects that represent each application option.

Example
<tcw-omnibar title-text="Title text">
  <tcw-omnibar-app-launcher-button slot="action">tcw-omnibar-app-launcher-button>
tcw-omnibar>
Angular example
<tcw-omnibar title-text="Title text">
  <tcw-omnibar-app-launcher-button slot="action" [optionsCallback]="appLauncherOptionsListener">tcw-omnibar-app-launcher-button>
tcw-omnibar>
import { Component, OnInit } from '@angular/core';
import { IAppLauncherOptionsCallback, IAppLauncherOptionsResult } from '@tylertech/tyler-components-web';

@Component({
  selector: 'app-launcher-example',
  templateUrl: './app-launcher.Angular.html'
})
export class AppLauncherExample implements OnInit {
  public appLauncherOptionsListener: IAppLauncherOptionsCallback;

  public ngOnInit(): void {
    this.appLauncherOptionsListener = () => this._getAppLauncherOptions();
  }

  private _getAppLauncherOptions(): Promise {
    // In a real world application you would make an HTTP request here, and map the result
    // into an IAppLauncherOptionsResult type (structure shown below)
    return new Promise(resolve => {
      resolve({ 
        options: [
          // This array contains objects of type IAppLauncherOption (should match the default intent type from your intents service)
        ]
      });
    });
  }
}

API details

As noted above, the app launcher button component is provided for convenience to ensure consistency and ease of use when integrating the omnibar into an application. Below you will find an example, and the API for this component.

Properties/attributes

Property

Attribute

Description

optionsCallback

A reference to a function that the app-launcher uses to fetch its options asynchronously.

Methods

Name

Description

focusFirst()

Sets focus to the first option.

Events

Name

Description

tcw-app-launcher-select

Emits when an option in the app-launcher popup is selected. This allows for the ability to prevent the default action of opening the URL in a new tab, and handling the action manually.

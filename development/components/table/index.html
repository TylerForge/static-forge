Table

The table component renders itself based on the configuration that is provided to it. It does not accept any child HTML elements to define it’s rows and/or columns. The table renders in the light DOM so that it can accept global styles for any custom templates that are provided to it.

Installation

This component can be installed as an individual npm package via @tyler-components-web/table

To install the full TCW component library package, see the getting started guide.

Guidance

The table component is used to display tabular data. When combining this component with the Paginator component you can display a single page of the full data set.

Usage

The table component requires the use of JavaScript to interact with it. It handles the rendering itself for ease of use, while being flexible in the functionality it provides.

Using the component in the HTML markup is simple. Just add the following HTML:

<tcw-table>tcw-table>

This alone won’t render anything, but with a little configuration from JavaScript we can setup a table with data easily.

In the example code below, we are getting a reference to the tcw-table custom element, and then we set the data and
columnConfigurations properties.

When those two properties are both set, the table will render the rows of data.

const table = document.querySelector('tcw-table');
table.data = [
  { Id: 1, Name: 'First1 Last1', Description: 'Test description 1' },
  { Id: 2, Name: 'First2 Last2', Description: 'Test description 2' },
  { Id: 3, Name: 'First3 Last3', Description: 'Test description 3' }
];
table.columnConfigurations = [
  { property: 'Name', header: 'Person name', sortable: true, initialSort: true },
  { property: 'Description', header: 'Person description' },
  { property: 'Age', header: 'Person age', sortable: true, align: 'right' }
];

Using column filters

To add column filters you will need to specify a filterDelegate for each column that needs it. You can use the built-in component delegate classes (see example below) for the common filter input types, or you can build your own delegate class that represents the component you would like to render in the filter cell.

A component delegate is a flexible API that allows you to define a class that extends an AbstractComponentDelegate. This class defines a common interface to use for dynamic components that require the ability to get/set values, set invalid, disabled, required… etc. You can then communicate with your component through this delegate instance.

import { TextFieldComponentDelegate, SelectComponentDelegate } from '@tylertech/tyler-components-web';
import { SomeCustomComponentDelegate } from './my-custom-component-delegates';

table.columnConfigurations = [
  { property: 'name', header: 'Person name', filter: true, filterDelegate: new TextFieldComponentDelegate({ placeholder: 'Filter name...' }) },
  { property: 'color', header: 'Favorite color', filter: true, filterDelegate: new SelectComponentDelegate({ placeholder: 'Filter color...' }) },
  { property: 'age', header: 'Person age', filter: true, filterDelegate: new SomeCustomComponentDelegate() }
];

Angular usage

Below is an example usage of interacting with the tcw-table component within an Angular application.

This first example shows how to set up the bindings within an Angular template:

<tcw-table [columnConfigurations]="columnConfigurations"
           [data]="data$ | async"
           select="true"
           select-key="Id"
           (tcw-table-select)="onSelect($event)"
           (tcw-table-select-all)="onSelectAll($event)"
           (tcw-table-sort)="onTableSort($event)">
tcw-table>

The following snippet is an example component class with the properties/methods to handle inputs/outputs on the tcw-table element. The data property is being used an RxJS observable here, but by using the async pipe we are able to unwrap the observable to provide an array of objects to the table component.

@Component({
  selector: 'table-example',
  templateUrl: './table-example.component.html'
})
export class TableExampleComponent implements OnInit {
  public columnConfigurations: IColumnConfiguration[] = [
    { header: 'Name', property: 'Name', sortable: true, initialSort: true },
    { header: 'Age', property: 'Age', sortable: true },
    { header: 'Position', property: 'Position', sortable: true }
  ];
  public data$: BehaviorSubject;

  public ngOnInit() {
    // Note: In a real application this data would likely be set from an async data source
    const players: IPlayer = [
      { Id: 1, Name: 'Tom Brady', Age: 41, Position: 'QB' },
      { Id: 2, Name: 'Julian Edelman', Age: 32, Position: 'WR' },
      { Id: 3, Name: 'Rob Gronkowski', Age: 29, Position: 'TE' },
      { Id: 4, Name: 'Chris Hogan', Age: 30, Position: 'WR' },
      { Id: 5, Name: 'James White', Age: 26, Position: 'RB' }
    ];
    this.data$ = new BehaviorSubject(players);
  }

  public onSelect(evt: CustomEvent): void {
    const data = evt.detail as ITableSelectEventData;
  }

  public onSelectAll(evt: CustomEvent): void {
    const data = evt.detail as ITableSelectAllEventData;
  }

  public onTableSort(evt: CustomEvent): void {
    const data = evt.detail as ITableSortedEventData;    
  }
}

Properties/Attributes

Property

Attribute

Description

data

The collection of data to bind to the table.

columnConfigurations

The column configuration objects that define how each data item is displayed.

select

select

Toggles the checkboxes in the first column of the table.

multiselect

multiselect

Whether to allow selecting multiple rows via the checkboxes (if select=true).

selectKey

select-key

The unique property name for each row in the data for keeping track of selections. This is required if select/multiselect is on.

dense

dense

Toggles whether the table rows use dense styling or not.

filter

filter

Toggles whether the table filter row is displayed.

fixedHeaders

fixed-headers

Controls whether the table headers stay fixed when scrolling. Default is false.

layoutType

layout-type

Controls the table-layout style on the table element. Valid values are "auto" (default), "fixed".

Methods

Name

Description

hideColumn(columnIndex)

Hides a column by its index.

showColumn(columnIndex)

Shows a column by its index.

selectRow(data)

Selects a row by providing a reference to the row data object.

selectRows(data, preserveExisting)

Selects a row by providing a reference to the row data object.

deselectRow(data)

Deselects a row by providing a reference to the row data object.

deselectRows(data)

Deselects multiple rows by providing a reference to the row data objects.

clearSelections()

Clears all selected rows.

render()

Forces the table to re-render as if it was initially connected.

expandRow(rowIndex, template)

Expands a row using the provided row index and template builder function.

collapseRow(rowIndex)

Collapses an expanded row by its row index.

isRowExpanded(rowIndex)

Returns whether a row is expanded by providing the row index.

getSelectedRows()

Returns the currently selected row instances.

Events

Name

Description

tcw-table-initialized

Emitted when the table has been initialized after connected.

tcw-table-select

Emitted when a table row is selected using the checkbox. The data is available through the event.detail property and is of type ITableSelectEventData.

tcw-table-select-all

Emitted when the select all checkbox changes.

tcw-table-select-double

Emitted when a row is double clicked.

tcw-table-sort

Emitted when a sortable column changes its sort direction.

tcw-table-filter

Emmited when a key is pressed in a table filter input.

Getting Started

To get started using the web components in a vanilla JavaScript applicaiton, follow the installation and setup steps below. For help developing your own web component library, refer to the Component Development guide.

If you’re looking for a guide that is specifically geared toward Angular, please head over to the Angular Getting Started guide.

Step 1: Installation

Before continuing, make sure you have read the prerequisites page to ensure that your system is prepared to work with the TCW libraries.


$ npm install --save @tylertech/tyler-components-web --registry http://yarpvproget/npm/npm_Tyler_Core/

Step 2: Usage

The package contains a pre-built webpack bundle that can be directly used on a webpage via script tags, or you can import each component individually as needed if consuming in a module loading environement via the provided ES modules. 

Now that you have the package installed, you need to ensure that you define all components that you are using. This is done via the exported definition functions provided by each individual component, or the whole library can be defined via the convenience function called defineComponents.


import { defineComponents } from '@tylertech/tyler-components-web';
defineComponents();

If you are consuming the library in a environment that does not support module loading, add a script tag that references the library:


<script type="text/javascript" src="node_modules/@tylertech/tyler-components-web/dist/tyler-components-web-lib.min.js">script>

Now you can call the defineComponents function from the global TylerComponentsWeb variable:


TylerComponentsWeb.lib.defineComponents();

Step 3: Global StyleSheet

While most components do contain their styles and are standalone web components, various components in this library rely on a global StyleSheet to be in place. These components do not use Shadow DOM style encapsulation. The StyleSheet exists within the @tylertech/tyler-components-web package in the dist folder. You can reference the file via a script tag in the head section of your web page:

rel="stylesheet" href="node_modules/@tylertech/tyler-components-web/dist/tcw-material-design.css">

Step 4: Fonts and Icons

Add a link tag that references the Roboto font (with desired font weights), and MaterialIcons font to allow for properly setting up our typography and iconography:


rel="stylesheet" href="//fonts.googleapis.com/css?family=Roboto:400,500,700,400italic|Material+Icons">

Note: We are currently referencing these assets through Google’s public CDN. This will change in the future when we set up our own Tyler CDN to host these assets.

Step 5: Custom Elements ES5 Adapter

The custom elements specification requires that all custom elements be defined as an EcmaScript class that extends HTMLElement. The TCW library includes bundles that have been compiled down to ES5, which means that the custom element classes have been downleveled to a JavaScript function type. To make these custom element functions compatible with browsers that expect them to be a class, we need the additional step of including a polyfill from the @webcomponents/webcomponentsjs package.

First you need to install the package in the project:


$npm install --save @webcomponents/webcomponentsjs

Now you can import the adapter or include it in your page via a script tag. The only requirement is that the file be loaded before any of your custom elements.


import '@webcomponents/webcomponentsjs/custom-elements-es5-adapter.js';

Or

          
<script type="text/javascript" src="node_modules/@webcomponents/webcomponentsjs/custom-elements-es5-adapter.js">script>;

Although the default TCW code is downleveled to ES5, the ES6 bundles are still provided if you only need to target browsers that natively support custom elements. In this case the adapter is not required.

Step 6: Polyfills (optional)

If you need to support browsers that do not natively support custom elements, the additional step of including polyfills is required.

Ensure that you have the polyfills installed:


$ npm install --save @webcomponents/webcomponentsjs

Now you can either import or use a script tag referencing the polyfills as you did with the ES5 adapter in the previous step


import '@webcomponents/webcomponentsjs/webcomponents-bundle.js';

Or


<script type="text/javascript" src="node_modules/@webcomponents/webcomponentsjs/webcomponents-bundle.js">script>;

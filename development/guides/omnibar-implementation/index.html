Omnibar implementation

This is a guide that is intended to provide general information and detailed implementation details regarding the TCW omnibar and its sub-components. You will learn about design decisions and how to integrate these components into your application.

Introduction

This guide will help walk you through the process of integrating an application with the TCW omnibar web components. If you have further questions/concerns please provide feedback directly to the Forge team via our integrated feedback form, or on slack under the #tyler-forge channel.

Design decisions

There have been a few iterations of omnibar in the past. It has undergone style changes, functional changes… etc. but there has always been a need to adjust it slightly on a per-application basis. Previously this required including the functionality within the component itself, further bloating its scope. 

The intention going forward is to provide a set of components that can be “composed” to create the experience you are looking for. Instead of providing all functionality into a single component, the omnibar has been split up into various individual components that can be used together. This helps keep component responsibilities lean, while still providing the consistency and flexibility that Tyler applications need.

Components

Let’s start by introducing you to the various omnibar-related components that are available:

Omnibar

The omnibar component itself is just a simple component the provides common styles, structure, and typography. It is the most basic implementation that you can get. By itself, it’s nothing more than a container with a background color, logo and title. The power of using this component starts to shine when you look at it from the stand point of consistency. Every Tyler application should have a consistent header. The elements within that header can differ slightly, but overall this component will provide that base consistency.

<tcw-omnibar title-text="Title text">tcw-omnibar>
Menu button

The menu button is nothing more than a predefined icon button. This component ensures that every application that needs to display a menu icon button will do so in the same way. It can be placed within the menu slot of the omnibar component.

<tcw-omnibar title-text="Title text">
  <tcw-omnibar-menu-button slot="menu">tcw-omnibar-menu-button>
tcw-omnibar>

Search

This component is an input element that is styled to look natural within the omnibar. The intention is that this component will be placed in the center slot, and used to provide global and/or local searching within an application.

<tcw-omnibar title-text="Title text">
  <tcw-omnibar-search slot="center">
      <input type="text" placeholder="Search..." />
  tcw-omnibar-search>
tcw-omnibar>

You can listen for the tcw-omnibar-search event on the component to know when the user has executed the search.

Note that the input element is slotted in and left exposed to allow for easily binding to it using standard event listeners and/or various framework binding syntax without having to proxy all events and properties of the element through the Web Component.

To view API details for the omnibar search component see here.

App launcher button

The app launcher button is also a simple component that provides a consistent icon button, while providing access to the internal app-launcher component. The app launcher component is responsible for rendering intents for applications that are registered. This component should be placed in the action slot.

<tcw-omnibar title-text="Title text">
  <tcw-omnibar-app-launcher-button slot="action">tcw-omnibar-app-launcher-button>
tcw-omnibar>

To view API details for the app launcher button see here.

Profile button

The profile button offers a common user avatar, and profile card within a popup. The intention of this component is to display the logged-in user information, as well as provide them the ability to (optionally) sign out. This component should be placed in the profile slot.

<tcw-omnibar title-text="Title text">
  <tcw-omnibar-profile-button slot="profile" avatar-text="First Last" full-name="First Last" email="first.last@tylertech.com">tcw-omnibar-profile-button>
tcw-omnibar>

To view API details for the profile button see here.

Help button

The help button component just provides a consistent icon button with access to populating options within the menu that is displayed when clicked.

<tcw-omnibar title-text="Title text">
  <tcw-omnibar-help-button slot="action">tcw-omnibar-help-button>
tcw-omnibar>

To view API details for the help button see here.

Notification button

The notification button currently just provides a consistent icon button, and facilities to show the notification count. Listen for the click event on the element to show your notifications if necessary.

Note: The goal in the future is to provide a common notification popup with enhanced functionality, but the infrastructure to support this across Tyler does not exist yet…

<tcw-omnibar title-text="Title text">
  <tcw-omnibar-notification-button slot="action">tcw-omnibar-notification-button>
tcw-omnibar>

To view API details for the notification button see here.

Implementation

The following sections will discuss the implementation details regarding each of the omnibar components noted above by providing code snippets, best practices, and expectations for each in multiple technologies.

Angular

Each of the omnibar components integrates seamlessly with Angular. Below you will find some common usage examples for these components in Angular:

Disclaimer: The following examples are overly-simplified on purpose to condense the content and convey the purpose and usage of each component. It is expected that the actual data would come from configuration and/or asynchronous services at runtime.

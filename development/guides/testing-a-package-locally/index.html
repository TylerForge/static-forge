Testing Locally

This guide will describe the process of using npm link to test the production build of your TCW component library locally on your machine inside of another project, such as an Angular application for example.

Build the npm package

Use the tcw CLI to build your component library, and generate a distributable npm package.

$ tcw build

Create a link for your package

Navigate to the dist/release/[package name] directory in your command line terminal.

$ cd dist/release/@tylertech/tyler-components-web

Run the following command to create a global link for your package on your machine.

$ npm link

Consume your linked package

To “install” your package within another package, such as an Angular application for example, run the following command from the root of your project.

$ npm link [package name]

Replace [package name] with the name of your package, $ npm link @tylertech/tyler-components-web for example.

The package should now be installed in your project as a symbolic link. You can use the package as if it were installed from the npm registry.

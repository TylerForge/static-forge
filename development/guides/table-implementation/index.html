Table implementation

This guide is intended to describe how to implement the TCW table Web Component, and answer questions about its structure, API, and usage.

Introduction

This guide will help walk you through the process of implementing TCW table Web Component in an application. If you have further questions/concerns please provide feedback directly to the Forge team via our integrated feedback form, or on slack under the #tyler-forge channel.

General information

The table component is intended to be an easy-to-use component that takes care of common boilerplate features, while being configurable and flexible enough to support various needs.

Most applications need to display data in a tabular format in one way or another. Typically, applications need to provide zero or more of the following features:

  • Sorting
  • Filtering
  • Paging
  • Editing
  • Row selection
  • Fixed headers
  • … etc.

To avoid having to implement each of these features manually, and to provide consistency in the presentation, these features are built-in to the table component in a configurable format (off by default). This helps provide strict consistency and familiarity when using the same component across applications and products.

Usage

This component is 100% configuration-based. It is intended to solve the “80% use case” that Tyler applications need. The component handles rendering of the rows and columns, and any other additional features based on the configuration provided to it. By default, the component will simply render a basic table with no interactive features.

While new features are going to be added as needed, if there is a need for your application to do something unique or unsupported by the component, it is always possible to create a standard HTML table element and style it using the CSS classes provided by TCW to gain the proper look-and-feel. This also opens the door for using third-party table components (Angular CDK table for example) while keeping the look-and-feel consistent.

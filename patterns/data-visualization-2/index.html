What chart should I use?

Comparing data

Over time

When displaying categorical data, or different types of data (e.g. dogs and cats) with the goal of showing that data in relation to time (‚Äúhow the population of cats and dogs in NYC has changed over the past 40 years‚Äù), use a line graph.

Between groups

When highlighting the differences between two groups, e.g. the population of dogs and cats in NYC, use a bar chart to help the user make a comparison. Stacked bar charts can be useful for comparing data between groups as well.

Displaying distributions

When the goal is to show the overall pattern (the forest vs. the individual trees) or distribution of data, use scatter plots, bubble charts, alluvials, histograms, and maybe maps or heatmaps. Whatever visually shows the pattern clearly is best.

Displaying subsets of data

When the goal is to¬†show the relationship between one of the data sets and the rest‚Ää‚Äî‚Ääwhere the individual data sets are a subset of a larger category,¬†use stacked bar charts, treemaps, or stacked area charts.¬†Pie charts can be appropriate here but should be used mindfully.

For example: to show a chart summarizing last year‚Äôs budget, each of the categories (rent, utilities, gas, etc.) is a subset of the budget year, so a stacked bar chart would work well.

Bar charts

Bar charts present categorical data with rectangular bars with heights or lengths proportional to the values that they represent. The bars can be plotted vertically or horizontally. Bar charts should be used to compare different values that are hierarchically equivalent.

Use for:
  • Observing differences in trends over time
  • Showing an overall trend (line charts are less good at drilling down on each value for a point in time)
  • Showing comparisons
  • Showing correlations
Use with:
  • Time-based data
  • Categorized data
Don't:
  • Don‚Äôt compare between values that have different units (cost in one category and time in another, both on the y axis).

  • Showing correlation

Stacked bar charts

A standard bar chart compares individual data points with each other. Stacked bar charts allow users to see changes in a series of data and where they occurred, especially in comparison to the totals. They can be great alternatives to pie charts.

Use for:
  • Comparisons
  • Show correlations
  • Explore in time
  • Show subdivisions of data‚Äìtwo y values per each x value.
  • Alternatives to pie charts
Use with:
  • Time-based data
  • Categorical data
  • Use the stacked bar instead of a regular bar chart if you want to show a total or projected amount alongside the value.¬†
Don‚Äôt:

Don‚Äôt use when the focus is on comparing the sizes of the individual categories or when the total sum of the elements in the bar is not relevant.

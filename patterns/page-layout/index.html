There are several core page layouts that we depend on in UX to build our applications. Using them breeds consistency, clarity, and efficiency in our applications.

Application entry point: List view

Single line listÂ 

Ideal for 3-5 attributes

Faceted card list

Ideal for 3-5 primary data elements, with a primary header such as a name or record number + the desire to expand for mid-level detail â€“ additional 3-5 ancillary data elements, ability to drill into a detail view.

Data table

Ideal for 4-8 data elements, use for financial data or when column sort is desired.

Implementation of each

Single line list:

  • Add new fab + speed dial for contact 3-5 most common contact types (Mediator, etc.)
  • Edit (new screen with back arrow)
  • Add new and edit are the same form, one view only, one editable
  • Search

Faceted card list:

  • Name as header, 3-5 primary contact data elements displayed on card
  • Next 3-5 contact data elements displayed on click to expand
  • Click again to drill into full contact detail, with ability to invoke edit (pencil icon)
  • Add new fab + speed dial for contact 3-5 most common contact types (Mediator, etc.)

Data table:

  • Sort on header columns
  • Search on header row take over (can include bulk actions, i.e. select all and change all contacts from single authentication to 2-factor authentication, activate, deactivate, etc.)
  • Edit can be an inline dialog which folds down from the baseline for editing a single data element at a time or could be a modal dialog for multi-element editing. Any edits over 3-4 pieces of data should be handled via full screen rather than modal.
  • Add new fab + speed dial for contact 3-5 most common contact types (Mediator, etc.)

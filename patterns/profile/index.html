Profile

Profile patterns display personal information about people: citizens, employees, managers, customers.

Guidance

Profiles can be displayed as the following patterns:

  • Mini contact card (small)
  • List results (small)
  • Profile card (medium)
  • Full page profile (large)

Mini contact card

Use a mini contact card as a “quick reference” to a specific person. Mini contact cards include:

  • an avatar (image or avatar component)
  • first and last name 
  • primary identifying information 
  • secondary information (optional)
  • primary actions or chips (optional)
Default contact card
Full contact card

List results

Use list results to display people in the context of a search. Profiles can be displayed as 

  • list items 
  • task cards
Default task card

Use a default task card to allow users to expand for more detail and to take up to two actions. Default task cards should not navigate to a detail page

Full task card

Use a full task card to display a person’s name and up to five supporting pieces of data. Full task cards may be static or may navigate to a detail page.

List results

Use a list view to display a person’s name, an additional piece of information (optional) and an action (optional).

Profile card

Use a contact card to display information about people at a medium size. 

Profile card with image

Use when images are available and are likely to be used to identify a specific person. Best used in cases where a user is familiar with people by face (a teacher viewing his students, a manager viewing her immediate team).

  1. Image
  2. Name & primary identifier (profession, address, etc)
  3. Secondary information (may be displayed as a body content, label-value pairs, or an icon list)
  4. Primary actions (up to two)
  5. Notifications (optional)
  6. Overflow menu (optional, contains additional actions)
Additional layouts

Secondary information may be displayed as label-value pairs, or as an icon list. 

Profile card without image

When images aren’t likely to be available for people, use the following pattern. 

  1. Avatar
  2. Name & primary identifier (profession, address, etc)
  3. Secondary information (may be displayed as label-value pairs or an icon list)
  4. Primary actions (up to two)
  5. Overflow menu (optional, contains additional actions)
Additional layouts

Secondary information may also be displayed as an icon list.
